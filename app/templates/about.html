About PlanSowGrow

Mission

PlanSowGrow is a garden planning and tracking application built with clean architecture principles and modern web development best practices.

Technology Stack
  • Backend: Python 3 + Flask
  • ORM: SQLAlchemy
  • Database: MariaDB
  • Frontend: HTML + Bootstrap 5 + Jinja2
  • Architecture: Clean Architecture with service layer
Design Principles
Separation of Concerns

The application is organized into distinct layers:

  • Models: Database entities and relationships
  • Services: Business logic and validation
  • Routes: HTTP request handling
Business Logic in Services

All business logic resides in the service layer. Routes are kept thin and only handle HTTP concerns, delegating all business operations to services.

Historical Data Preservation

The system uses soft delete patterns to ensure historical data is never permanently removed. Records are marked as deleted rather than removed from the database.

Date and Period Management

Dates and periods are core concepts in the system. Built-in mixins provide consistent handling of time-based entities like seasons and plantings.
